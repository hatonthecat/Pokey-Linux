Tiny X Linux



This linux distro is designed to be VERY compact yet give most of the basic functions of an Xwindows Linux system. It should be compatible with most kernel2.2/4/libc2.1 distros (you can just drop in binarys from slackware/suse/mandrake). It's really just for fun, but maybe a start point for developers who want to distribute their linux apps to ms-windows users, or people who just want to download a very quick demo of what linux can do.

Why?

Well I really just did it for a bit of fun as I felt most distros were too big, and thought it would be good to have a 2 disk xwindows system that was flexible and would run most recent software.

It's not really meant for:
corporate users to reuse old PC's as X terminals (running X on any thing less than a 486dx is not advisable)
Or:
using as an embedded OS for handhelds or settop boxes
Or:
using as a system rescue disk

But it could probably be used for most of these tasks with only minor modification

It is really just my attempt to create a absolute bare minimum linux distro. Using the tools available you should be able to do most things, with a little thought you can use chimera with shell scripts to create many gui based applets. you can browse the web, paint pictures, run a terminals, manage the filesystem,

In the good old days of unix all the tools were small, extensible, scriptable, and pipable. Unfortunatly things are rapidly moving towards the huge monolithic application structure that hogs resources, as users become moronic button clickers rather than programmers. I hope with this distro to use only tools that follow the unix ideal yet integrate well into Xwindows apps. To give basic tools and allow a really compact system that can do almost anything, if you can write shell scripts

Features

It takes less than 4Mb compressed or 6Mb installed and contains the following tools:

  • kernel2.4.12(now this is totally standard)/libc2.1
  • X11R6-4.1(hacked kdrive)
  • busybox
  • ppp chat and pppd
  • iptables support
  • udhcp - a small but untested dhcp client/server
  • chimera web browser(hacked version will replace FvwmScript)
  • xpaint (hacked version command line opts,pixmaps, other minor tweaks)
  • xfm(hacked pixmaps may integrate with chimera)
  • alloywm (a small and quite sane wm)
  • xedit(I may hack this or find a better editor)
  • compiled from debian2.2.3/mandrake7.2

    Support

    This is Beta code so you can consider in testing, It works on my system but may not work on yours!
    Sorry But I wont be supporting anything, you can send me your problems but I wont answer unless I find them particularly interesting or they help by patching something.
    If you have problems with something it would probably be best to get intouch with the author of that component. If X crashes then contact XFree86.org, if the system wont boot try posting to one of the linux kernel newsgroups. obviously you must give listings of any error messages and what led upto the crash occuring. I have put some links to info about the components of the system below. If you want support for something in the kernel then then you will have to compile the appropriate module/kernel from the linux kernel source.
