43. MMM Browser

MMM was a web browser created in 1995 by a French developer and was implemented in Objective Caml (Categorical Abstract Machine Language). Its interface was called CamlTk41 which used Tcl/Tk. It supported HTML 2.0, most of HTML 3.2 and frames. A disclaimer on the project's web site reads, "WWW surfing is wasting your time. Writing WWW pages is wasting my time. But we both are probably too bored to do anything else at this moment, aren't we? This page has been hit far too many times. Please have mercy". The last version released was in December 1997.

44. Mosaic

NCSA Mosaic was one of the first web browsers which came out in January 1993. It was instrumental in popularizing the World Wide Web and the Internet in general by integrating multimedia such as text and graphics. Mosaic was developed at the National Center for Supercomputing Applications (NCSA) at the University of Illinois at Urbana–Champaign beginning in late 1992. Microsoft licensed Mosaic to create Internet Explorer in 1995. By 1997 Mosaic had lost the browser wars to both Netscape Navigator and Internet Explorer. Mosaic's last release was in January 1997. Other web browsers of note based on Mosaic are mMosaic and Mosaic-CK.

45. Mnemonic

Project Mnemonic released its web browser in February 1998 by decrying "feature bloat" in the more common web browsers of the era. The project was an effort to build an extensible, modular, standards compliant web browser available under the GNU general public license. The project was shot-lived however, as the final release for Mnemonic was in July 2000.

46. Netomat

First available in June 1999, Netomat was called "The Non-Linear Browser". When Netomat was running, random visuals and snippets of sentences mined from the Web floated endlessly across a black backdrop, accompanied by clips of sound if the user desired. It was used as a platform for spawning countless browser interfaces that you or anyone could create. A user could create fully interactive multimedia emails, website and blogs. The content was limited only by your imagination. Linux versions were made availble up through April 2001. Windows and Mac versions were released up to January 2004.

47. Netrick

Created in 2001, Netrick was a simplistic text mode web browser, similar to w3m and Lynx, but it was rather limited compared to other text mode browsers of the day. Netrick began to lag behind in development, and by the latter half of 2002, the project was abandoned.

48. NetRaider

NetRaider was a small and fast web browser first made available in late 2000. It was based on the KHTML engine that Konqueror used. It did not need any environmental libraries installed to run it. There were 2 alpha releases in December 2000, but by March 2002 the netraider.de website no longer listed the project.

49. OSB Browser

OSB was released in October 2004 by the Nokia Research Center as part of ongoing internet browser-related research activities. It was an example browser implementation which used Gtk+ Webcore and KHTML. OSB's main purpose was to demonstrate rendering engine capabilities and to test porting status. The last release was in March 2007.

50. Plume

Created by an Australian developer in 1996 and originally called Surfit!, it was advertised as Tcl Meets The World Wide Web. It could post HTML forms for CGI scripts and could do HTML tables but not frames. With the IMG extension for Tk, it could handle most image types found on the web. The project didn't last long however, as June 1997 was its last release.

51. Pyleon

Pyleon was first announced in July 2001. An alpha release on SourceForge.net was available from a CVS repository. Pyleon was an extensible GTK+ web browser written in Python, with support for both Gecko and Gtk HTML. Non-core browser features were implemented as modules so power users could load only their desired features and programmers could easily add their own. Pyleon development was abandoned in February 2005 due to limitations in pygme and lack of pyxpcom documentation.

52. QtWeb

First released in September 2008, QtWeb has been described as a lightweight web browser. It used the WebKit engine that was embedded in the Qt framework. Features included adblock, BitTorrent download manager, bookmarks including import and aliases, customizable keyboard shortcuts, customizable user agent, pop-up blocking and tabbed browsing. It was criticized however for the lack of extension support, its inability to run Java and its problems working with many websites. The last release was in September 2013.

53. Red Baron

Distributed by Red Hat version 4 in October 1996, Red Baron was simply described as a secure web browser. It was proven to be very buggy however. Red Hat 4.2 in May 1997 was the last time it was offered.

54. rekonq

rekonq was a lightweight, QtWebKit-based web browser released in December 2008 by the KDE project. Its aim was to be a standalone and simple web browser. It had features such as tabbed browsing, adblock, proxy support, and support for Flash and Java plugins. The project was eventually abandoned, as its last release was in January 2014.

55. retawq

Released in 2001, retawq was an interactive multi threaded text-based browser. It was small, fast and easily configurable. You could keep open as many "virtual windows" as you wanted and work simultaneously in two of them in split-screen mode. retawq was proven to work with SlimLinux (a small floppy disk-based system); ttylinux (which ran on i486 computers with as little as 24 MB of RAM) and X Windows on a Floppy (Xwoaf). The user interface was based on Ncurses. March 2006 was its last release.

56. Scamper

Scamper began in 2006 as a simple web browser which ran in the Squeak programming language. Enough functionality was supported to browse the web. It did not support tables, cookies or javascript. After several years another version was released in May 2017, with a minor update in September 2018.

57. Sielo

Sielo is an open source browser made with Qt and WebEngine that was released in September 2017. A new design called Tab Spaces allows the user to divide their Sielo window into several spaces, as if you had several windows next to each other but still interacting with each other. There's also a Floating Button feature which gives you access to all the needed controls during your browsing and can adapt itself to your workspace. A newer version of Sielo, called sielo-vision was released in November 2020, but no further updates have been applied and the browser should be considered experimental and not fully up-to-date.

58. SkipStone

First made available in 2000, SkipStone was a Gtk+ web browser that used Mozilla's engine Gecko. It aimed to be light and fast with fewer dependencies. Support for WebKit was then added. It spent a number of years in development but eventually the project was abandoned. SkipStone's last release was in April 2008.

59. Symphony
Symphony is currently in alpha, first released in June 2011. The developer claimed it was already very user-friendly and feature intensive. It used intuitive features and designs which resulted in a very simplistic web browser. No other updates have ever been given for this browser.

60. StarOffice

StarOffice, also known as StarSuite, originated in 1985 as StarWriter. This office suite could also be used to surf the web starting with version 5 in 1998. StarOffice was purchased in 1999 by Sun Microsystems to compete with Microsoft Office. From a consumer standpoint, StarOffice was essentially the parent to OpenOffice and the grandparent to LibreOffice. The first OpenOffice version in 2002 did not include the web browser feature. Ultimately, Sun was acquired by Oracle in 2010. That version was briefly known as Oracle Open Office before being discontinued in 2011.

61. Tavia

Tavia was a KHTML web browser released in March 2004. It had multi-document interface support and was similar to Konqueror. Tavia never got past alpha stage however.

62. UbuMonkey

Released in October 2010, UbuMonkey's goal was to create a stable, modifiable and light web browser for the Linux desktop. It had Flash support and simple customizable themes, but lacked the ability to have bookmarks and an adblocker. The last release was in May 2011.

63. Uzbl

Uzbl became available in April 2009 as a web interface of tools which included a web browser. It was text-based and controlled exclusively by the keyboard. Minor revisions were added periodically over several years. Implementation of WebKit2 was a major goal of the project, but this never came to fruition. The final version came out on GitHub in October 2016.
